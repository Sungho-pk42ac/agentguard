file="a;rm -rf /" and line=99. resolveEditor() honored AGENTGUARD_EDITOR=code,

with no shell:true, so ';' 'rm -rf /' is never re-parsed by any shell; it is
inert text baked into a single array element. See (d3) for direct proof against
the shipped dist module.
VERDICT: PASS — hostile path is inert; no shell interpretation is possible.
