# AX evidence acceptance receipt

## purpose

한국어 우선 AX Rollout Guard 심사/리뷰 상황에서 `PR diff`, `MCP config`, `transcript/log`, `SARIF/report`, `smoke manifest` 증거를 한 장짜리 acceptance receipt로 묶는다. 목표는 “이 증거를 누가 받았고, 어떤 조건이면 승인/보류/차단 판단을 내릴 수 있으며, 언제 다시 실행해야 하는가?”를 30초 안에 설명하는 것이다.

이 문서는 fixture-backed docs-contract slice다. CLI behavior, scanner rules, severity, verdict semantics, `PASS` / `REVIEW` / `BLOCK`, rule IDs, JSON, SARIF, API, machine fields는 현재 계약을 유지한다.

## fixture-backed evidence commands

아래 command는 fresh clone에서 `npm ci && npm run build` 후 실행하는 것을 전제로 한다. 위험 fixture는 의도적으로 `BLOCK` 또는 `REVIEW`를 만들 수 있으며, 그 non-zero exit은 evidence 생성 실패가 아니라 risky input 판정일 수 있다.

```bash
node dist/index.js scan-diff < examples/risky-pr.diff
node dist/index.js scan-mcp < examples/risky-mcp.json
node dist/index.js scan-log --policy examples/agent-policy.yaml < examples/agent-transcript.log
node dist/index.js scan-diff --sarif --out .agentguard-demo/acceptance-receipt.sarif < examples/risky-pr.diff
npm run smoke:ax-demo
```

Required fixture/artifact paths:

- `examples/risky-pr.diff`
- `examples/risky-mcp.json`
- `examples/agent-policy.yaml`
- `examples/agent-transcript.log`
- `.agentguard-demo/acceptance-receipt.sarif` (generated, not committed)
- `scripts/ax-demo-smoke.mjs`
- `.agentguard-demo/ax-evidence-smoke/manifest.json` (generated by `npm run smoke:ax-demo`)

## machine-contract boundaries

- `PASS`, `REVIEW`, `BLOCK` remain English-compatible verdict values.
- `scan-diff`, `scan-mcp`, `scan-log`, `doctor`, JSON, SARIF, API, `ruleId`, `artifactLocation`, and `region.startLine` remain machine-facing contracts.
- `agentguard scan-diff`, `agentguard scan-mcp`, and `agentguard scan-log` command names stay stable in Korean docs.
- Human Markdown can be Korean-first, but automation-facing output and schemas stay stable for CI, code scanning, and parsers.

## claim guardrails

- no fake adoption: 이 receipt는 실제 고객사, 레퍼런스 고객, 도입 완료, 또는 운영 인증을 주장하지 않는다.
- no certification/parity: OWASP, MCP, GitHub, SARIF, Snyk, or any public reference is not presented as endorsing, certifying, approving, replacing, or being equivalent to AgentGuard.
- no runtime auth claim: this card does not claim runtime OAuth enforcement, runtime authorization enforcement, session binding, redirect URI validation, token validation, consent enforcement, or MCP server execution control.
- no automatic SARIF upload: SARIF command creates a local artifact; upload, triage, workflow permissions, and reviewer channel ownership remain separate owner decisions.
- no behavior change: this receipt does not change scanner rules, blocking policy, CLI commands, exit-code semantics, package publishing, hosted auth, or dashboard behavior.
