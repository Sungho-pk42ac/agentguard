AI agents changed the threat model.

A coding agent is not just a code generator. It can read files, run commands, call tools, write PRs, and connect through MCP to services that were never designed for autonomous use.

The blind spot is behavioral.

Static application security testing can inspect the code that lands in the repository. AgentOps security asks a different question: what did the agent touch on the way there?

  • Secrets can move through logs. A transcript or shell command may expose values that never appear in source.
  • Diffs hide operational intent. A PR can look small while the agent used risky commands to produce it.
  • MCP expands reach. Tool scopes can grant access to GitHub, Slack, Drive, databases, or internal APIs.

AgentGuard fits into the review loop you already run.

The scanner is intentionally small and deterministic: pipe in the evidence from your agent workflow, review the finding, and keep the human decision at the merge gate.

01 Capture the surface. Pipe PR diffs, agent logs, transcript text, workspace files, and MCP configuration into the CLI.
02 Scan for agent risk. Detect secrets, PII, dangerous commands, full-access tools, and policy exceptions.
03 Route the result. Emit markdown, JSON, or SARIF for PR comments and GitHub code scanning.
